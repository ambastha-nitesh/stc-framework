Compliance & Audit Integrity

SEC 17a-4
WORM Audit Backend

Append-only audit with rotation-seal records. Refuses prune_before and erase_tenant with ComplianceViolation. Pair with S3 Object Lock or Linux chattr +a for real WORM.

  • Rotation-seal records bridge file boundaries
  • Marker file detects out-of-band directory swaps
  • STC_AUDIT_BACKEND=worm to enable
HMAC-SHA256
Tamper-Evident Chain

Every record chained via HMAC-SHA256 keyed by STC_AUDIT_HMAC_KEY. An attacker with file access but not key access cannot forge. Replaces the prior plain-SHA-256 chain that was tamper-evident only against accidents.

  • key_id stamped per record for rotation
  • verify_chain(accept_unknown_genesis=True) for post-prune
  • Ephemeral-key fallback only in dev; prod refuses
ed25519
Signed Spec Verification

The spec YAML is the compliance posture. An ed25519 signature sidecar (spec.yaml.sig) must verify against STC_SPEC_PUBLIC_KEY at startup. Prod refuses to boot with an unsigned or invalid spec.

  • Sign from HSM / Yubikey / sigstore in prod
  • sign_spec() helper for CI fixtures
  • Spec hash recorded in every system_start audit
Per-event retention
Retention by Regulatory Class

AuditSpec.retention_policies replaces the single retention_days. Erasure receipts and boundary-crossings default to 6 years (FINRA 4511); chain-seal records to forever.

  • Pruning refused when any class is "forever"
  • Compliance events survive naive retention overrides
  • Per-class defaults in RetentionPolicy
GDPR Art. 17
Right-to-Erasure, Cache-Aware

erase_tenant() now clears idempotency cache, budget samples, rate-limit bucket, vector docs, history, tokens, and (on GDPR backends) audit rows. Previously-cached responses cannot resurface post-erasure.

  • Erasure receipt written under tenant_id=None
  • Chain recomputed after audit erasure
  • CLI: stc-governance erase <tenant> --yes
GDPR Art. 15 / 30
DSAR Export + Audit of the Audit

export_tenant_records() walks every tenant-scoped store and returns a single DSAR artifact. The export itself is audited (dsar_export event) so the regulator sees proof of fulfilment.

  • Tenant-scoped via vector store filters
  • CLI: stc-governance dsar <tenant>
  • 26-event AuditEvent catalogue

Production Safety & Observability

Fail-closed prod
Strict Prod-Mode Startup

STC_ENV=prod enforces six invariants at astart(): HMAC key set, tokenization strict, no content logs, no mock LLM, WORM backend, signed spec. Missing any raises STCError and the K8s readiness probe fails.

  • Defaults ship as dev-safe, prod-unsafe
  • One explicit opt-out per check
  • Kubernetes never routes traffic to a misconfigured pod
TOCTOU-safe
Atomic Budget Reservation

Per-tenant costs tracked in calendar-day buckets with a monotonic-clock sanity check. reserve() holds the lock across check-and-book to close the race where concurrent requests both pass enforce then both charge.

  • O(1) observed() per window
  • Wall-clock jumps cannot corrupt buckets
  • Refund on crash + block_input paths
At-least-once safe
Idempotency Cache

LRU + TTL cache keyed by (tenant_id, idempotency_key). Repeated requests return the cached QueryResult without re-charging, re-auditing, or re-counting. Cleared on tenant erasure.

  • Client retries are first-class
  • No double-billing on transient failures
  • Flask headers + library API both honour it
RPS-bounded
Per-Tenant Rate Limiter

Token-bucket per tenant with bounded memory (max-tenant cap + LRU eviction). Budget caps spend; rate limit caps request rate — a tenant with cheap queries can't flood the system.

  • Retryable rejection (clients back off)
  • STC_TENANT_RPS + burst knob
  • Cleared on erasure like every tenant store
Drain-on-SIGTERM
Graceful Shutdown

SIGTERM triggers a coordinated drain: _stopping flag rejects new requests, InflightTracker.wait_idle blocks until pending queries complete, adapters flush via aclose(), audit log closes last.

  • Default 30-second drain (configurable)
  • K8s preStop hook compatible
  • No partial audit records on rolling deploys
Adapter health
Deep /readyz Probe

ahealth_probe() calls healthcheck() on every adapter (LLM, vector, embeddings, prompts) with a short timeout. /readyz returns 503 when any fails — K8s evicts the pod before traffic lands.

  • stc_adapter_healthcheck gauge
  • Strict-health startup option fails fast
  • Presidio warmed at astart()

Security Hardening

ReDoS-safe
Bounded PII Regex

Every PII pattern has bounded quantifiers and no overlapping alternations. A 10 KB adversarial payload to the redactor terminates in milliseconds, not seconds.

16 rule families
Expanded Prompt-Injection Rails

Multilingual overrides (DE/ES/FR/IT), zero-width smuggling, base64-encoded payloads, chat-markup tokens, role-prefix spoofing, delimiter breakout, translation exfiltration. Defence in depth: output rails also scan for reflected injection.

Indirect-injection
Chunk Sanitisation

Every retrieved document chunk is redacted + sanitised before reaching the LLM. A poisoned document cannot inject via RAG, and a chunk containing blocked PII is dropped rather than leaked.

Defence-in-depth
Data Sovereignty, 3 Layers

Restricted-tier routing enforced at spec load, at set_routing_preference(), and again at dispatch. Removing any one layer is a compliance regression. Bedrock-in-VPC recognised as in-boundary.

Header-safe
Log Injection Defence

Every header value is CR/LF/NUL/control-char stripped before it enters a log line or audit record. Hostile clients cannot forge additional log entries via X-Tenant-Id: tenant1\r\n....

Supply chain
CI: SBOM + Secret Scan + CVE

CycloneDX SBOM artifact on every run, pip-audit --strict, regex-based secret scan (AWS keys, Slack tokens, private keys, OpenAI keys). New deps require review.

Audit chain, visualised
